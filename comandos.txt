docker build --tag <NOMBRE_TAG>  . >ubicacion> construira la imagen descargandose los recursos necesarios
docker images > nos listara las imagenes guardades en docker
docker tag prueba-node:latest prueba-node:v1.0.0 > agregamos una etiqueta a la ultima version y esto sirve para mantener un historia de imagenes en nuestro docker

docker rmi prueba-node:latest > eliminamos la imagen

docker run --publish PUERTO_EXTERNO:PUERTO_INTERNO NOMBRE_IMAGEN > correr la imagen exponiendo un puerto en particular

si a run le ponemos --detach o --d hara que se ejecute en segundo plano

docker ps > listara los contenedores > un conjunto de imagenes que se estan ejecutando o no

docker ps --all > muestra la lista del historial de imagenes ejecutados y sin ejecutar

docker restar ID| NAME > reiniciar un contenedor que no esta iniciado

nota : un contenedor puede tener mas de una imagen y es el encargado de ejecutar las imagenes

docker exec --interactive --tty CONTAIER_ID COMAND_UNIX < sirve para ejecutar un comando en el contenedor, los comandos son basados en linux
